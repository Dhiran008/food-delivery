# 🍔 Food Delivery Database System

## 📌 Project Overview
This project is a Food Delivery Database Management System developed using MySQL. 
It simulates a real-time food ordering platform like Swiggy/Zomato where users can place orders, make payments, and track deliveries.

---

## 🎯 Objectives
- Design a normalized relational database
- Manage users, restaurants, orders, payments, and deliveries
- Implement automation using triggers
- Optimize queries using indexes
- Practice advanced SQL concepts

---

## 🗂️ Database Tables

1. users
2. restaurants
3. menu_items
4. orders
5. order_items
6. payments
7. delivery_partners
8. deliveries

---

## 🔗 Relationships

- One User → Many Orders
- One Restaurant → Many Menu Items
- One Order → Many Order Items
- One Order → One Payment
- One Order → One Delivery
- One Delivery Partner → Many Deliveries

---

## ⚙️ Features Implemented

✔ Primary & Foreign Keys  
✔ Joins (Inner, Left)  
✔ Subqueries  
✔ CTE  
✔ Window Functions (RANK, ROW_NUMBER)  
✔ Aggregate Functions  
✔ Stored Procedures  
✔ Triggers  
✔ Index Optimization  
✔ Revenue & Analytics Queries

## 🚀 Advanced Concepts Used

- Composite Index
- Audit Trigger for order status changes
